# 版本记录

这个项目用 Git tag 标记已经封装的正式版本，用 Git branch 标记仍在生长的草稿版本。当前正在使用的是第三版草稿；第二版只是已经封装的历史版本。

## v1-prototype

对应提交：

```text
cf0b192 Initial personal blog
```

这是第一次提交的个人博客原型。它已经包含 Astro 静态前台、本地后台、文章、专栏、资料存档、图书馆、搜索索引和 GitHub Pages 部署工作流。

这个版本的意义是：确认整个产品方向成立，即“公开前台 + 本地后台 + GitHub Pages 发布”。

## v2-current

对应第二版封装点：

```text
e5eeb59 Document project versions
```

第二版在原型基础上补齐了更接近日常使用的能力：

```text
站点名称改为“无边落夜”
日间/夜间主题切换
文章管理页支持编辑、删除、首页推荐和发布时间倒序
专栏管理支持排序和删除专栏标签
资料存档支持外部链接和 Markdown 原文
图书馆后台按最近编辑优先，前台按最早添加优先
中文标题自动生成拼音 slug
保存后给出手动 git commit + push 指令
公开页面自动检测新版部署，减少旧页面缓存问题
Cloudflare Worker + D1 记录文章浏览量，后台可见
```

这个版本的意义是：从“能展示的博客”进入“能长期维护的个人知识库”。

## v3-draft

对应分支：

```text
v3-draft
```

这是当前正在使用的第三版草稿，不创建 tag，也不发布 Release。它会持续接收第二版之后的新功能和修复；等确认第三版需要封装时，再把当时的草稿状态转换为正式 tag 和 Release。

当前已经进入第三版草稿的变化包括：

```text
发布后保留 Git 推送提示
修复上传图片在前台和后台预览中的显示
图片改为“编辑时预览，保存/发布时导入项目”
远程图片 URL 保存/发布时本地化到 public/uploads/
保存草稿后保持在当前编辑页面
未手动编辑 slug 时，slug 持续跟随标题更新
```

这个版本的意义是：把博客从“能长期维护”继续推进到“更可靠的内容资产管理”。
